TouchinginvokeTauri, remote-runtime.ts, llm-api streaming, http_server.rs, http_dispatch.rs, local-file-api, Tauri plugins, file dialogs, managed asset URLs, imports/uploads, native window/titlebar behavior, opener behavior, haptics, Spotify OAuth, TTS, or Docker/server build files.
ReviewDoes the feature require desktop-only native access, or can it safely run through the hostable runtime data directory and JSON/SSE HTTP contract? Is the command allowlisted in remote-runtime.ts and handled explicitly in http_dispatch.rs or a dedicated route? Does the feature avoid raw server filesystem paths and silent no-op fallbacks?
Verifypnpm typecheck, pnpm build, cargo check --manifest-path src-tauri/Cargo.toml --workspace, and an HTTP smoke check when practical: run marinara-server, hit /health, then exercise the supported shared API path from Settings > Advanced > Remote Runtime URL.
DecisionIf remote behavior cannot be equivalent, keep the feature desktop-only and document that limit instead of adding silent fallbacks.

Shared Change Risk

engine/contracts

Review schemas, DTO consumers, shared API adapters, and any persisted records.

engine/generation-core

Review chat, roleplay, game, agents, prompt preview, and replay behavior.

src/shared/api

Review all features calling the adapter and the matching Tauri command DTO.

Hostable runtime

Review remote allowlists, HTTP dispatch, asset URL resolution, uploads, streaming, and native-plugin limits.

engine/mari

Review Professor Mari UI, Mari API wrapper, Rust command request shape, and provider/tool support.

src/shared/components

Review every feature surface using the component, including mobile layout.

If the impact area cannot be named clearly, the change is probably in the wrong owner or too broad for one commit.
